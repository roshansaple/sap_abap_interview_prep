For a **3-year experienced SAP ABAP Developer**, interviewers usually expect both strong ABAP fundamentals and practical project experience. They often ask scenario-based questions rather than only definitions.

## 1. Data Dictionary (DDIC)

**Questions:**

* What is the difference between a Transparent Table, Cluster Table, and Pooled Table?
* What is the difference between a Domain and a Data Element?
* What are Foreign Keys and Check Tables?
* What is a Table Maintenance Generator (TMG)?
* What is the difference between Database View, Projection View, and Maintenance View?
* How do you enhance a standard table?

**Expected:** You should explain with project examples.

---

## 2. Internal Tables

**Questions:**

* Difference between Standard, Sorted, and Hashed tables?
* When would you use a Hashed Table?
* What is the difference between READ TABLE and LOOP AT WHERE?
* What is a Secondary Key?
* How do you improve performance while processing large internal tables?

**Scenario:**

> You have 1 million records and need to find one record quickly. Which internal table would you use and why?

## 12. Coding Questions

### Remove Duplicates

```abap
DELETE ADJACENT DUPLICATES FROM itab COMPARING matnr.
```

### Find Highest Salary

```abap
SORT it_emp BY salary DESCENDING.
READ TABLE it_emp INDEX 1.
```

### Join Two Tables

```abap
SELECT a~vbeln
       b~posnr
  FROM vbak AS a
  INNER JOIN vbap AS b
  ON a~vbeln = b~vbeln
  INTO TABLE @DATA(lt_data).
```
