🟢 PROJECT 1 — OO ALV REPORT

I developed a Sales Order Line Item Report using OO ALV.

Requirement:

Business users needed a structured report to analyze sales order item data.

What I Did:

I developed a screen-based OO ALV using custom container and field catalog.
Data was fetched from VBAP based on selection criteria.

I also implemented conditional row coloring to highlight high-value items above 12,000.

Outcome:

This helped users easily identify important transactions and improved data analysis.

🟢 PROJECT 2 — ODATA SERVICE

I worked on developing a custom OData service for Sales Order data.

Requirement:

Business wanted to create and retrieve sales orders through API-based integration.

What I Did:

I created custom header and item tables and implemented CRUD operations.

I also handled deep insert for creating header and item data together.

Outcome:

This enabled smooth integration and real-time data access.

🟢 PROJECT 3 — MODULE POOL

I developed a Module Pool application for data maintenance.

Requirement:

Users required a guided interface for safe data entry.

What I Did:

I built screens using PBO and PAI logic and added validation before database updates.

Outcome:

This reduced incorrect data entries and improved usability.

🟢 PROJECT 4 — BADI ENHANCEMENT

I implemented a BADI enhancement in Purchase Order processing.

Requirement:

Business wanted to restrict PO creation for blocked vendors.

What I Did:

I used ME_PROCESS_PO_CUST to validate vendor block and GST details.

Outcome:

This ensured compliance and prevented incorrect PO creation.

🟢 PROJECT 5 — CDS VIEW

I created a CDS view for monitoring Purchase Order values.

Requirement:

Business needed visibility into high-value POs.

What I Did:

I built CDS on EKKO and EKPO with calculated fields and integrated it with ALV.

Outcome:

This improved reporting performance using code pushdown.
